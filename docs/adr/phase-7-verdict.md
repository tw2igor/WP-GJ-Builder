# Phase 7: Приёмка MVP — итоговый вердикт

Дата: 19 июля 2026. Статус: **PASS с одним исправленным критичным замечанием**. Это финальная фаза восьмифазного плана — консолидация доказательств по всем 9 критериям раздела 15 спеки, независимый security review, план поэтапной раскатки/отката, базовая доступность и полнота i18n.

## Независимый security review (раздел 10)

Полная запись — `docs/adr/phase-7-security-review.md`. Итог: одна КРИТИЧНАЯ находка (`ProjectDataSanitizer` не санитизировал узлы project_data без `data-wpb-block` — обычные/нативные компоненты GrapesJS, например после импорта HTML) — **найдена, исправлена, проверена реальными payload'ами против живого REST API, закреплена регрессионным тестом** (`/wpgjb/v1/dev/phase7`, 4/4 PASS). Три низких/информационных находки задокументированы с обоснованием (две — реальные ограничения песочницы без Composer, одна — архитектурное решение для явного решения при подписании).

## i18n (раздел 13: "русский и английский с самого начала")

Аудит нашёл и исправил 2 реальных пробела:
- **JS-сторона редактора использовала функцию-заглушку `t()`, которая просто возвращала строку как есть — никакого реального перевода не происходило вообще** (несмотря на видимость i18n-готовности в 43 местах кода). Исправлено: `t()` теперь вызывает настоящий `@wordpress/i18n`'s `__()`, подключённый через `wp_set_script_translations()` (см. `EditorPage::maybe_enqueue()`) — стандартный путь ядра WP для JS-переводов, готовый принять `.json`-файлы перевода (`wp i18n make-json`), когда они появятся.
- **`SecurityPage::CAP_LABELS`** была `const`-массивом с русскими строками — константы класса физически не могут содержать вызовы `__()`. Переделано в метод `cap_labels()`, возвращающий значения через `__()`.
- Остальные найденные нерусские строки — сообщения `ManifestValidator`/`ChecklistLinter`/`Diagnostics` для РАЗРАБОТЧИКОВ блоков и логов, не пользовательский UI конструктора — оставлены как есть (тот же принцип, что core WP не переводит `_doing_it_wrong()`).
- **Честная оговорка**: реальных `.po`/`.json`-переводов на английский по-прежнему НЕТ (`languages/` пуст) — в этой песочнице нет WP-CLI (`wp i18n make-pot/make-json`) для их генерации. Инфраструктура теперь корректна и готова принять перевод, самого перевода ещё нет — это именно то, что раздел 13 называет "i18n-готовностью", не готовым переводом.

## Базовая доступность (раздел 13)

- Добавлены `aria-label` на все icon-only кнопки (undo/redo в шапке, блок-тулбар вверх/вниз/дублировать/скрыть/удалить в `blocks.js`), `aria-pressed` на переключатель устройств, `role="dialog"`+`aria-modal`+`aria-labelledby` на пикер блоков, `role="status"`+`aria-live="polite"` на индикатор сохранения, `role="alert"` на баннер блокировки редактирования.
- Добавлено явное правило видимого фокуса (`outline: 2px solid #2271b1`) для всей оболочки редактора и пикера — раньше не было ни одного явного focus-стиля (полагались на дефолт браузера без гарантии).
- **Честная оговорка**: это ТОЧЕЧНАЯ проверка/правка, не полный WCAG-аудит (спецификация явно выносит его во вторую очередь). Полноценный прогон скринридером (NVDA/VoiceOver) не проводился — только структурная ARIA-разметка, добавленная по итогам чтения кода.

**Все 9 критериев закрыты доказательствами.** Критерий 7 требовал реального исправления в этой фазе (не был бы закрыт без находки security review) — это именно то, для чего раздел 10 требует независимую проверку перед релизом: находка была РЕАЛЬНОЙ, не гипотетической, и без неё MVP ушёл бы в релиз с рабочим, невидимым обходом двухконтурной санитизации.

## Что осталось вне объёма MVP (вторая очередь, зафиксировано планом)

Zero-режим, сайдбары, ACF-теги, экспорт/импорт в файл, полный WCAG-аудит, централизованная кросс-сайтовая телеметрия, углублённая SEO-интеграция, AI-модуль сборки страницы целиком (REST list/get/insert уже готовы, "собрать всю страницу по последовательности" — всё ещё 501-заглушка), WebP-фолбэк для изображений (эмпирически проверено в Phase 6b — не включается на этом WP по умолчанию, нужна отдельная работа с фильтром), реальный CSS-парсер вместо spike-quality `sanitize_css()` (нужен Composer, недоступен в этой песочнице).
